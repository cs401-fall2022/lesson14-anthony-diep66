Introduction

JavaScript was originally made as a simple scripting language for websites. Small snippets of Javascript were used to introduce user interaction with the website and anything more than that was considered unusual. As time went on and with the massive growth of "the web", Javascript became increasingly popular among developers. What started as small, embedded javascript code hidden inside browsers evolved into applications such as node.js being created.

Javascript could however use some improvements. For one, Javascript produces some rather unexpected behaviors when different types of literals are compared. 

                    if( "" == 0 ){

                        //returns true here

                    }

Perhaps for someone that programs in C++ or Java, this is rather unusual as the compairison amongst differing types of variables are being compared, that is, a string literal to a number literal. Another odd quality of Javascript are the accessing properties of objects 

                    const obj = {
                        x: 10;
                    }
                    
                    //successfully executes, undefined
                    obj.printSomething();

One last point to make is Javascript's pure dynamic typing. Given the intentionally vague Javascript code: 

                    message.toLowerCase();
                    message();

Here, there are a couple of things to consider:

  1. Is message callable?
  2. Is toLowerCase() a function of message?
  3. If so, is toLowerCase() a callable function?

Due to Javascript's dynamic typing, the interpreter can't answer any of these questions. Perhaps message was actually a string. In that case, message() certainly wouldn't make sense. The only way to find out is to run the code and see what happens, there are no static type checking available.

Now, introducing Typescript, a superset of Javascript that introduces static type checking and enforces a stricter set of rules towards the usage of types. Division by zero, invalid array access, and the previously mentioned unusual behaviors of Javascript are all addressed in Typescript.

Types

This idea extends to creating objects or interfaces with inferred types

Example with Object

const user = { name: "Hayes", id: 0, };

Example with inferred types

interface User{ name: string; id: number; }

Composing Types

There are 2 ways of composing types:

Unions:

type MyBool = true | false;

Generics:

type NumberArray = Array<string>

Structural Types

Typescript also has a thing called duck typing where objects with the same shape are considered to be the same type 

                        interface Point{
                            x: number;
                            y: number;
                        }

                        function logPoint(p: Point){
                            <!--
                                do something 
                            -->
                        }
                        const point = {x: 12, y: 26;}
                        logPoint(point)

Here, calling logPoint on the point objected that was created will not throw any errors and TypeScript recognizes the point object to be of the Point interface type.
